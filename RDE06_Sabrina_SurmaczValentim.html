O QUE É SOFTWARE DE VERSIONAMENTO?

Sistema que tem por objetivo realizar o controle de versões de um código ou documento.
É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.
O software de versionamento é também conhecido como software de controle de versões.

ENTENDENDO PARA QUE SERVE

Quando vamos desenvolver um projeto, como a escrita de um documento Word, podemos proceder assim...
Pegamos um notebook, criamos uma pasta e um arquivo com o projeto. Feita a entrega, pode ser que ocorra a necessidade de uma correção ou atualização. Assim, criamos uma nova versão do arquivo para fazer as correções.
Porém, essa maneira não é muito adequada, já que, por exemplo, após as correções, salvamos o arquivo, se arrependemos, mas a versão anterior não está mais disponível.

Para não acontecer tais problemas, utilizamos o software de controle de versões, o qual, vai criar um repositório local e um histórico de versões.

Além do repositório local tem o repositório remoto.

COMO FUNCIONA?

Tem um servidor específico para trabalhar com o repositório remoto, o software de controle de versões irá gerenciar o histórico de versões local e remoto, fazendo a sincronização para que tudo o que está no repositório local esteja no remoto e vice-versa.

Se gerenciar versões para um usuário já é difícil, imagine para dois...
Exemplo: há uma pasta compartilhada de um projeto em que diversos usuários acessam a pasta, inserindo e alterando arquivos. Se alguém apagar alguma coisa, ou fazer alguma alteração indevida, gera problemas, por isso é importante utilizar o software.

Os dois usuários trabalham com um mesmo projeto, o usuário A tem o projeto no notebook e um repositório local, utilizando o software.
O usuário B também tem o repositório local.
Quando eles fazem a sincronização com um servidor, utilizando uma plataforma para repositório remoto, o software de versionamento irá controlar as alterações, permitindo ou não que façam alterações, notificando os usuários.

TIPOS DE SOFTWARE DE CONTROLE DE VERSÕES

CENTRALIZADO
Possui um repositório central em um servidor com controle de versões.
Dependente do servidor.
Exemplo:Software Subversion.
DISTRIBUÍDO
Possui um repositório local com controle de versões.
Atualizações no repositório central.
Independe do servidor.
Exemplo:Softwares Mercurial, Git.

GIT

  • É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux, foi projetado e desenvolvido por Linus Torvalds (criou o Linux) em 2005.
  • Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central.
  • Sistema de controle de versões distribuído.
  • SIGNIFICADO DO NOME: Segundo Linus, ele é um bastardo egocêntrico que nomeou todos os projetos dele com o seu nome. O primeiro projeto foi o Linux e depois Git. Git é uma gíria para cabeça dura. Porém dizem que Git é a agregação de um sistema global de rastreamento.

GITHUB

  • Plataforma de hospedagem de código-fonte e arquivos com controle de versão.
  • Surgiu em 2008 e foi adquirido em 2018 pela Microsoft por US$7,5 bilhões.
  • O GitHub hospeda cerca de 100 milhões de projetos com 56 milhões de usuários.
  • O GitHub é a plataforma de hospedagem, já o Git é o programa que vai fazer o controle do versionamento.
  • Mascote: Octocat - Significa que pode ter vários braços no projeto e também pode ser feita uma junção (polvo-gato)
